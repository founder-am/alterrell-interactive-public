# ALTERRELL INTERACTIVE — PLATFORM BRIEF
Last updated: May 2026

## CORE PRINCIPLE
This platform is a structured publishing system, not a design experiment.

Everything exists to produce:
- fast mobile readability
- high-shareable narrative clarity
- repeatable production cycles

---

## ROLES (STRICT SEPARATION)

### AMA (OWNER)
- Writes original voice (essays, arguments, lived observation)
- Decides meaning
- Approves structure

### CHATGPT (STRUCTURE + EDITORIAL)
- Compresses AMA writing into:
  - tab systems
  - card systems
  - narrative flow maps
- Improves clarity and rhythm
- Does NOT rewrite voice unless asked

### CLAUDE (SYSTEM + BUILD ENGINE)
- Converts approved structure into:
  - HTML layouts
  - component scaffolds
  - visual specs
- Enforces design system
- Does NOT generate new arguments or voice

---

## CORE WORKFLOW

1. AMA writes long-form essay (Substack or draft)
2. ChatGPT converts into:
   - tabs
   - cards
   - visual plan
3. Claude Code builds:
   - HTML structure
   - layout implementation
   - visual placeholders
4. Final review by AMA

---

## NON-NEGOTIABLE RULES

- No system file overrides unless explicitly updated
- No invention of new narrative structure by Claude
- No rewriting AMA voice in build phase
- No design changes without platform approval

---

## OUTPUT PRIORITIES

1. Mobile readability first
2. Scan behavior (headline → stat → insight)
3. Shareability second
4. Visual polish last

---

## DESIGN SYSTEM

- alterrell-interactive.css is canonical
- never rebuild it
- only extend via local <style> blocks

---

## CLAUDE LIMIT

Claude must operate as:
- transformer (not author)
- builder (not strategist)
- enforcer (not designer)
